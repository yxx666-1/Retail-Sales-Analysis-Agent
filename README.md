# 零售销售数据智能分析Agent
## 项目简介
本项目是基于大模型Agent技术开发的**零售场景智能数据分析系统**，旨在解决传统数据分析门槛高、需要专业SQL技能、数据解读不结合业务经验的痛点。用户只需输入自然语言提问，系统即可自动完成「SQL生成→数据查询→可视化绘图→业务策略输出」全流程，无需任何编程基础即可完成专业级零售销售数据分析。

项目基于LangChain框架搭建ReAct智能体，集成RAG检索增强能力，可结合本地业务分析文档输出可落地的运营建议，是面向零售行业的低代码、轻量化智能分析工具。

## 技术栈
| 模块 | 技术选型 |
|------|----------|
| 大模型 | 通义千问 qwen-turbo |
| Agent框架 | LangChain ReAct |
| 向量检索 | FAISS 向量数据库 + HuggingFace Embedding |
| 数据库 | SQLite 轻量关系型数据库 |
| 数据可视化 | Matplotlib |
| 开发语言 | Python 3.10+ |

## 核心功能
### 1. 自然语言转SQL（NL2SQL）
- 支持中文自然语言提问，自动识别查询意图、维度、聚合方式
- 自动生成标准SQL语句，查询零售销售数据库
- 支持多维度查询：地区、品类、客户类型、时间、销售额等
- 自动处理SQL语法错误、字段匹配问题

### 2. 自动化数据可视化
- 智能识别用户绘图需求，自动判断图表类型
- 支持柱状图、折线图、饼图三类常用图表
- 自动适配坐标轴、标题、中文显示，生成高清300DPI图片
- 支持销售额趋势、地区分布、品类占比等多场景可视化

### 3. RAG检索增强业务分析
- 基于FAISS向量库实现本地业务文档语义检索
- 结合真实销售数据 + 业务分析文档，输出专业运营策略
- 支持高销地区维护、滞销品类优化、客户分层运营等场景建议
- 避免大模型凭空编造策略，输出内容贴合实际业务

### 4. 多轮对话上下文关联
- 支持连续提问，上下文自动关联
- 可基于上一轮查询结果做进一步钻取分析
- 支持对比分析、趋势预判等进阶查询

## 快速开始
### 环境准备
- Python 3.10 及以上版本
- 通义千问API密钥（阿里云百炼平台免费获取）

### 安装依赖
```bash
# 克隆项目
git clone https://github.com/yxx666-1/Retail-Sales-Analysis-Agent.git
cd Retail-Sales-Analysis-Agent

# 安装依赖包
pip install -r requirements.txt
```

### 配置密钥
在项目根目录新建 `.env` 文件，填入你的通义千问API密钥：
```env
TONGYI_API_KEY=你的通义千问API密钥
```

### 运行项目
```bash
python agent_rag_sql.py
```
启动成功后即可在控制台输入自然语言问题进行查询。

## 效果演示
### 示例提问1：数据查询
> 输入：统计每个地区总销售额，从高到低排序
```
🔍 生成SQL：SELECT Region, SUM(Sales) AS Total_Sales FROM sales_data GROUP BY Region ORDER BY Total_Sales DESC
✅ 查询成功，数据：
 Region  Total_Sales
   East    702618.63
   West    680100.66
Central    492527.80
  South    392151.46

📊 最终分析结果：
东部地区销售额最高，达到70.26万，占整体销售额的31%；其次是西部地区68.01万，占比30%；中部和南部相对较低，分别为49.25万和39.22万。
建议重点维护东部市场，加大库存补给，同时针对中西部市场制定针对性的促销活动，提升整体销售额。
```

### 示例提问2：自动绘图
> 输入：生成各个品类销售额的柱状图
```
🔍 生成SQL：SELECT Category, SUM(Sales) FROM sales_data GROUP BY Category
🖼️ 图表已生成：各个品类销售额.png
📊 最终分析结果：
科技品类销售额最高，其次是家具类，办公用品类最低。建议重点推广科技品类产品，针对办公用品类制定捆绑销售策略，提升整体营收。
```

### 示例提问3：RAG策略输出
> 输入：东部销售额最高，应该怎么制定运营策略？
```
🔍 生成SQL：SELECT Region, Category, Segment, SUM(Sales) FROM sales_data WHERE Region='East' GROUP BY Region, Category, Segment
📊 最终分析结果：
东部地区核心营收来源为科技品类个人客户，运营策略建议：
1. 库存优化：加大科技品类、家具类产品的东部仓库库存占比，避免缺货
2. 客户分层：针对企业客户推出专属折扣方案，挖掘B端客户潜力
3. 活动运营：季度性推出满减活动，拉动办公用品类产品销售
4. 地域深耕：在东部核心城市开设线下体验店，提升品牌渗透率
```

## 项目架构
```
用户自然语言提问
    ↓
大模型意图识别 + SQL生成
    ↓
SQLite数据库查询 → 返回结构化数据
    ↓
识别绘图需求 → Matplotlib自动生成可视化图表
    ↓
RAG检索业务文档 → 匹配相关运营经验
    ↓
大模型整合数据 + 文档 → 输出完整分析结论与策略
```

## 项目亮点
1. **零代码门槛**：无需掌握SQL、Python，纯自然语言即可完成专业数据分析
2. **端到端闭环**：从提问到数据、图表、策略一站式输出，无需人工中间处理
3. **业务贴合度高**：RAG检索增强避免大模型幻觉，策略可直接落地执行
4. **轻量化部署**：基于SQLite轻量数据库，无需复杂环境配置，单机即可运行
5. **可扩展性强**：支持对接更多大模型、新增数据源、扩展更多分析工具
